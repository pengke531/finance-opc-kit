# F02 Data Collector - Agent Responsibilities

## 🎯 核心职责

### 1. 数据收集
- **实时数据**: 股票价格、指数、期货、外汇等实时行情
- **历史数据**: K线数据、历史价格、交易量等历史数据
- **财务数据**: 财报数据、财务指标、业绩预告等
- **市场数据**: 行业数据、板块数据、市场情绪数据
- **新闻数据**: 公司新闻、行业新闻、政策新闻等

### 2. 数据验证
- **准确性验证**: 交叉验证多个数据源
- **完整性检查**: 检查数据是否有缺失
- **异常值检测**: 识别和处理异常数据
- **时效性验证**: 确保数据是最新的
- **一致性检查**: 验证相关数据的一致性

### 3. 数据整理
- **格式转换**: 转换为标准化的数据格式
- **指标计算**: 计算技术指标和衍生指标
- **数据清洗**: 去除错误数据，填充缺失值
- **数据聚合**: 按时间、行业等维度聚合数据
- **数据存储**: 高效存储和管理数据

### 4. 数据服务
- **请求响应**: 响应其他Agent的数据请求
- **主动推送**: 主动推送重要的市场数据
- **数据报告**: 生成数据质量报告和统计报告
- **数据可视化**: 提供数据图表和可视化
- **历史查询**: 支持历史数据查询和分析

## 📋 工作流程

### 标准数据处理流程
1. **接收请求**: 接收来自其他Agent的数据请求
2. **解析需求**: 理解数据类型、时间范围、精度要求
3. **选择数据源**: 根据需求选择最佳数据源
4. **获取数据**: 从API或数据库获取数据
5. **验证质量**: 验证数据的准确性和完整性
6. **处理数据**: 清洗、转换、计算指标
7. **格式化输出**: 格式化为标准输出格式
8. **返回结果**: 返回数据给请求方

### 实时数据处理流程
1. **接收订阅**: 接收F05的实时数据订阅
2. **建立连接**: 建立实时数据连接
3. **接收数据**: 实时接收市场数据
4. **快速验证**: 快速验证数据合理性
5. **立即推送**: 立即推送给订阅方
6. **监控连接**: 监控连接状态，及时重连

### 异常处理流程
1. **检测异常**: 检测数据异常或源异常
2. **记录日志**: 详细记录异常情况
2. **尝试恢复**: 尝试恢复数据获取
3. **切换数据源**: 必要时切换到备用数据源
4. **通知相关方**: 通知受影响的Agent
5. **后续处理**: 异常解除后恢复正常服务

- ✅ **计算权限**: 可以计算技术指标和衍生数据

- ✅ **服务可用**: 确保数据服务的可用性

### 限制条件

- ❌ **不能超出范围**: 不处理非金融类数据

## 📊 工作标准

### 数据质量标准
1. **准确性**: 数据必须准确无误
2. **完整性**: 数据必须完整无缺失
3. **时效性**: 实时数据延迟<1秒，历史数据<5秒
4. **一致性**: 相同数据不同源必须一致
5. **可追溯性**: 数据来源和处理过程可追溯

### 服务质量标准
1. **响应速度**: 标准请求<2秒，复杂请求<10秒
2. **可用性**: 服务可用率>99.5%
3. **并发能力**: 支持10个并发请求
4. **错误处理**: 优雅处理所有异常情况
5. **用户体验**: 提供清晰的数据格式说明

### 性能标准
1. **处理效率**: 每秒处理>100条数据
2. **存储效率**: 高效压缩存储历史数据
3. **查询效率**: 历史数据查询<3秒
4. **计算效率**: 技术指标计算实时完成
